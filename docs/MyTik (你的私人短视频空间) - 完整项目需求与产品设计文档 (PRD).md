# MyTik (你的私人短视频空间) - 完整项目需求与产品设计文档 (PRD)

## 一、 项目定位与核心价值

- **产品愿景**：一款主打**绝对隐私**、**零数据采集**的本地短视频沉浸式管理与浏览工具。
- **核心痛点解决**：手机相册里存了大量搞笑、回忆、自拍的短视频，但相册浏览体验极差（需要频繁点击播放、退出）。MyTik 将这些本地视频转化为类似“抖音”的沉浸式连续滑动刷片体验。
- **超越原版（TikMy）的破局点**：
  1. **跨平台与多端演进**：原版为 iOS 独占，我们采用 `uni-app`，一期覆盖微信小程序，二期可直接编译为 Android/iOS 独立 App 以突破存储限制。
  2. **智能断链修复**：独创的数据导入恢复机制，即使本地视频源文件路径变更，也能通过特征哈希自动重连弹幕和分类数据。
  3. **本地推荐算法**：基于本地停留时长、点赞频次建立轻量级排序权重，并非简单的按时间倒序播放。
  4. **个人回忆弹幕**：用户针对某个视频发送的评论、弹幕与时间点都保存在本地；下次再次刷到同一视频时，仍可看到自己上次留下的内容，形成“私人回忆时间轴”。

## 二、 核心功能模块拆解

应用底部设三个 Tab：**库 (Library)、首页/播放 (Home)、设置 (Settings)**。

### 2.1 隐私与安全卫士 (Security)

*复刻原版截图中的“完全隐私”及密码盘功能*

- **产品边界调整**：考虑到微信小程序原生能力限制，首期默认**不实现隐私锁**，密码盘与生物识别均从首期范围中移除。
- **可选保留项**：如果后续验证发现微信端存在稳定、可接受的替代能力，再单独评估轻量锁屏方案；但不作为当前版本前置条件。
- **零网络请求承诺**：除小程序运行所需基础能力外，不在业务代码中发起任何云端数据库、视频上传或评论同步请求。

### 2.2 库与分类管理 (Library)

*复刻原版截图中的“智能分类”及网格试图*

- **自定义仓库（文件夹）**：
  - 支持创建自定义分类（如截图中展示的 "Love", "Puppy"）。
  - 每个文件夹卡片显示：自定义封面、分类名称、包含的视频数量。
- **多维度标签**：除了文件夹，视频可附加文本标签、描述，便于精准归类。
- **快捷悬浮导入**：在 Library 页面右上角提供常驻 `+` 号，点击快速呼出相册选择视频。

### 2.3 沉浸式刷片与单人互动 (Home)

*复刻原版截图中的“沉浸式刷片”、“弹幕评论”*

- **类抖音滑动交互**：
  - 全屏无缝上下滑动切换视频。
  - **性能优化限制**：使用 `swiper` + `video`。全局仅保留当前、上一条、下一条 3 个 `<video>` 实例，其余节点使用首帧截图（poster）占位，防止内存溢出导致小程序崩溃。
- **本地算法推荐**：系统记录单个视频的播放次数 (`playCount`) 和观看时长，自动提升喜爱视频的出现概率。
- **播放模式切换**：
  - 支持顺序浏览与随机浏览两种模式。
  - 用户可在播放器中切换“随机模式”，用于提升旧视频再次被刷到的趣味性。
- **单机弹幕/评论回显**：
  - 用户可在视频播放过程中发送自己的弹幕或评论。
  - 弹幕和评论只保存到本地，不存在任何他人互动与同步逻辑。
  - 当用户下次再次刷到同一视频时，系统会按原时间点展示自己上次留下的内容。
- **手势控制**：
  - 双击屏幕：触发点赞（心形动画）。
  - 长按屏幕：2.0 倍速播放。
  - 首期不实现双指捏合放大，避免与小程序视频组件能力和手势冲突产生不稳定体验。
- **手势可配置**：
  - 对易误触的关键手势提供开关或模式切换能力。
  - 重点控制“顶部区域触发回到第一个视频”这类高风险交互，默认关闭或替换为更安全的返回/跳转方式。
- **进度拖动**：
  - 在视频播放界面提供可见的进度条。
  - 用户可拖动进度条快速跳转到指定时间点，便于回看某一段内容或对应弹幕。
- **红心增强反馈**：
  - 允许一次操作触发多次红心动画，用于强化“很喜欢这个视频”的表达。
  - 多次红心主要作为本地交互反馈，不扩展为社交信号。
- **喜欢权重可调**：
  - 用户可调节“已点赞视频”的重复出现倾向。
  - 该配置会影响本地推荐排序，但不会改变原始视频数据。

### 2.4 单机回忆弹幕引擎 (Memory Danmaku - 核心)

*解决“本地沉浸式刷片”与“个人记忆留痕”的体验模块*

- **单人留痕机制**：
  - 用户可针对当前视频添加弹幕或评论。
  - 每条内容可携带视频时间点，用于回放时再次出现。
  - 所有内容只写入本地 JSON，不分享、不导出给他人、不合并外部评论。
- **再次刷到时自动回显**：
  - 当用户再次刷到相同视频时，系统自动读取该视频的本地弹幕记录。
  - 已保存的内容按时间点重新出现在播放器中，形成“和过去的自己一起看”的体验。
- **产品范围限制**：
  - 不做好友互动
  - 不做局域网互动
  - 不做异地同步
  - 不做 E2EE
  - 不做 WebSocket 中转

### 2.5 数据导入与导出引擎 (Data I/O)

*解决微信小程序本地存储痛点与数据安全的核心模块*

- **视频导入 (存入沙盒)**：
  - 调用 `wx.chooseMedia` 选择相册视频。
  - **防爆仓机制**：因小程序本地文件上限约 200MB，导入前需调用 `FileSystemManager.getSavedFileList` 检查容量。满载前给予预警。
  - 调用 `FileSystemManager.saveFile` 持久化到沙盒。
- **一键备份配置 (Export JSON)**：
  - 将所有分类、弹幕、点赞记录序列化为完整的 JSON 文件。
  - 可通过系统允许的方式导出或留作本地备份，作为用户的“资产备份”。
- **数据恢复与智能重连 (Import & Restore)**：
  - 用户导入备份的 JSON 文件覆盖本地设置。
  - **断链修复**：若换手机导致沙盒视频丢失，系统生成“待修复清单”，引导用户从相册重新选择对应视频，依靠时长和大小特征，自动将新路径与旧弹幕数据重新绑定。

## 三、 UI/UX 设计规范 (参考原版截图)

- **色彩系统**：
  - **主背景色**：极夜黑 `#080808` 或纯黑 `#000000`，提供极致的沉浸感。
  - **强调色**：荧光绿 `#34C759`（用于密码锁、选中状态、图标点缀）；心跳红 `#FF3B30`（用于点赞、重要警告）。
  - **文字色**：主标题纯白 `#FFFFFF`，副标题/时间戳浅灰 `#8E8E93`。
- **视觉元素**：
  - 摒弃复杂边框，大量使用高斯模糊（毛玻璃效果 `backdrop-filter`）作为弹窗和底部 TabBar 的背景。
  - 统一使用圆角卡片（`border-radius: 12px` 到 `16px`）。

## 五、 技术架构与演进路线 (降维打击策略)

**物理限制**：微信小程序的 `USER_DATA_PATH` 存在严格容量限制，且用户删除小程序会清空一切。

**演进路线建议**：

1. **第一阶段（验证期 - 微信小程序端）**：
   - 严格按照本 PRD 实施，优先实现“本地闭环”+“个人弹幕留存与回显”。
   - 定位为**“轻量级精选视频收藏夹”**。利用完善的“JSON 备份 + 视频源回溯重连”机制，弥补小程序易丢失数据的短板。
2. **第二阶段（体验完善期 - 小程序进阶）**：
   - 优化个人弹幕时间轴、回看体验、分类管理和恢复机制，提升单机沉浸感与可玩性。
   - 补充更多播放器偏好配置，例如手势开关、播放模式、喜欢权重调节等。
3. **第三阶段（完全体 - 独立 App 端）**：
   - 利用 uni-app 一套代码多端编译的特性，将项目打包为独立的 Android / iOS 应用。
   - **底层突破**：在 App 端，直接调用 `plus.io` 或 Native 插件，**直接读取系统相册的绝对路径进行播放**。无需再将视频复制到沙盒中，从而实现**零存储压力、瞬间导入、无上限的视频数量**。
   - 屏幕旋转与横屏沉浸式播放优先放在独立 App 阶段评估与实现，微信小程序端暂不作为首期需求。
