# MyTik 项目开发文档

## 1. 文档目的

本文档用于统一 MyTik 项目的开发方式、任务拆分、进度跟踪与测试协作方式。  
当前项目以 `docs/` 下现有文档为最高约束，本文件主要负责把“怎么开发、开发到哪一步、怎么验证”落到执行层。

适用范围：

- 微信小程序首期版本开发
- `uni-app + Vue 3 + TypeScript` 工程实现
- 单机模式、本地视频管理、个人弹幕回显
- 开发与测试并行推进

---

## 2. 当前项目目标

当前版本目标不是做社交互动产品，而是做一个**纯单机、沉浸式、本地优先的私人短视频浏览工具**。

当前核心范围：

- 本地视频导入
- 分类管理
- 沉浸式刷片
- 点赞与本地推荐
- 个人弹幕/评论留存
- 下次刷到时自动回显上次评论
- JSON 备份、恢复、断链修复

当前明确不做：

- 他人互动
- 局域网互动
- 异地同步
- 云端评论
- 首期隐私锁

---

## 3. 开发原则

- 以 `docs/` 文档为最高优先级
- 优先做最小可运行闭环
- 优先保证微信小程序端稳定可验证
- 优先本地数据安全和恢复能力
- 避免超前设计和无关重构
- 开发与测试并行，不等功能全部做完再统一验收

## 5. 推荐开发流程

### 5.1 总流程

每个功能按照以下顺序推进：

1. 阅读相关 `docs`
2. 明确当前功能的输入、输出、状态和边界
3. 先建立最小 UI 骨架
4. 接入本地状态与数据结构
5. 接入平台能力或本地文件能力
6. 同步补最小验证路径
7. 在微信开发者工具中进行页面级验收
8. 完成后更新本文件中的进度跟踪

### 5.2 开发与测试并行原则

每实现一个模块时，不等待“全部完成”再统一测试，而是立即进行对应验证。

并行方式：

- 页面骨架完成后：先做视觉和布局验证
- 本地状态接入后：立刻验证数据增删改查
- 文件导入接入后：立刻验证视频导入与容量边界
- 播放器接入后：立刻验证滑动、切换、回显和性能
- 备份恢复接入后：立刻验证导出、导入和断链修复

## 6. 模块拆分

### 6.1 Library 模块

目标：

- 展示分类卡片
- 提供导入视频入口
- 显示视频数量和封面
- 支持分类创建与管理

关键数据：

- `categories`
- `videos`

测试重点：

- 空状态是否正确
- 分类展示是否稳定
- 导入视频后数量是否同步
- 沙盒容量预警是否出现

### 6.2 Home 模块

目标：

- 沉浸式全屏滑动播放
- 点赞与播放统计
- 随机/顺序播放模式
- 个人评论/弹幕输入
- 再次刷到时回显历史评论

关键数据：

- `playCount`
- `totalWatchTime`
- `isLiked`
- `comments`
- 推荐排序配置

测试重点：

- `swiper + video` 三实例策略是否稳定
- 连续切换视频是否卡顿
- 弹幕时间点是否正确
- 再次刷到时历史评论是否正确回显
- 拖动进度条后弹幕是否仍与时间点一致

### 6.3 Settings 模块

目标：

- 播放模式配置
- 手势开关配置
- 喜欢权重配置
- JSON 导出与恢复入口

测试重点：

- 配置保存后是否持久化
- 设置切换后 Home 页面行为是否变化
- 导入备份后配置是否恢复

### 6.4 Data I/O 模块

目标：

- 视频导入到小程序沙盒
- JSON 备份导出
- JSON 导入恢复
- 视频断链修复

测试重点：

- 文件导入成功率
- 导入失败提示
- 备份内容完整性
- 恢复后视频与评论绑定是否仍正确

## 9. 测试策略

### 9.1 测试方式

当前阶段以手工测试为主，后续逐步补充自动化验证。

建议分层：

- 页面级测试：看 UI、交互、状态展示
- 功能级测试：看导入、播放、评论、恢复是否正确
- 边界测试：看容量、空数据、异常文件、路径变化
- 回归测试：改一个模块后检查其他模块是否被破坏

### 9.2 当前建议优先测试项

- 小程序启动后页面是否正常显示
- Library 页面空状态是否正常
- Home 页面切换是否正常
- Settings 页面配置是否能保存
- 构建产物是否能被微信开发者工具正确识别

### 9.3 后续关键测试项

- 导入多个视频后是否能稳定展示
- 多次切换视频后是否出现卡顿或黑屏
- 评论保存后再次刷到是否准确回显
- JSON 导出后再导入是否恢复完整
- 视频重新绑定后评论是否仍能匹配

## 10. 当前已知风险

- 微信小程序本地文件容量有限
- 视频组件在复杂手势场景下能力有限
- 横屏、双指缩放等能力首期不适合硬上
- 评论时间点与拖动进度条的同步逻辑需要谨慎设计
- 断链修复依赖轻量特征匹配，准确率需要真实数据验证

## 11. 需要的 skills / MCP / 工具

以下能力对后续开发最有价值：

### 11.1 已经很有用的能力

- `Filesystem MCP`
  - 用途：读写项目文件、维护文档、批量检查结构
  - 当前状态：已实际使用

- `Playwright / agent-browser`
  - 用途：如果后续跑 H5 预览、调试页面布局、验证交互流程，会很有帮助
  - 当前状态：可选，但后续 UI 验收阶段价值很高

- `nextjs`、`vercel` 类 skills
  - 当前项目不是 Next.js，不作为当前主能力

### 11.2 当前项目最建议用的 skills

- `playwright`
  - 当我们需要自动化验证 H5 预览页面、看布局和交互时使用

- `agent-browser`
  - 当需要通过浏览器自动检查开发态页面时使用

- `react-best-practices`
  - 当前项目是 Vue，不需要

- `shadcn`
  - 当前项目不是 React Web，不需要

### 11.3 当前项目最建议依赖的本地工具

- 微信开发者工具
  - 用途：小程序真实预览、能力验证、调试和验收
  - 重要性：高

- HBuilderX
  - 用途：作为当前 `uni-app` 项目的主要开发与快速预览工具
  - 重要性：高

### 11.4 当前开发链路补充说明

- 当前项目不再按“仅微信开发者工具开发”理解
- 实际开发链路为 `HBuilderX + 微信开发者工具`
- 推荐做法是先在 `HBuilderX` 中完成开发和初步预览，再在 `微信开发者工具` 中做小程序侧验收
- `appid missing` 当前不阻塞开发，暂不作为阶段性问题处理
### 11.5 如果你愿意补充，我最希望有的信息

- 微信开发者工具安装路径
- HBuilderX 安装路径
- 你的微信小程序 AppID 是否后续要固定到项目里

如果你把这几个信息告诉我，后面我可以更直接地帮你做联调和验收。
